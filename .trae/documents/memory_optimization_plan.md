# 内存优化计划

## 目标
优化内存使用，支持处理超大型项目

## 问题分析

### 当前内存使用场景
1. **文件内容读取**：`scan_file` 一次性读取整个文件内容到内存
2. **函数复杂度分析**：每个文件都需要存储完整的源代码和函数信息
3. **文件列表存储**：所有文件结果都存储在内存中
4. **字符串复制**：多处使用 `to_string()` 和 `clone()` 创建副本

### 优化方案

### 方案 1: 流式处理大文件
- 对于大文件，使用流式读取而非一次性加载
- 设置文件大小阈值（如 1MB），超过则分块处理

### 方案 2: 减少字符串复制
- 使用 `&str` 引用代替 `String` 复制
- 使用 `Arc<PathBuf>` 共享路径
- 避免不必要的字符串克隆

### 方案 3: 增量扫描
- 缓存已扫描文件的结果
- 检测文件修改时间，只重新分析修改过的文件

### 方案 4: 结果流式导出
- 扫描过程中直接导出结果，而非全部存储后再导出

## 实现步骤

### 步骤 1: 分析当前内存使用
- 添加内存统计日志
- 识别最大的内存占用点

### 步骤 2: 优化文件读取
- 在 `scanner.rs` 中添加大文件检测
- 实现分块读取（可选，复杂度较高）

### 步骤 3: 减少字符串复制
- 修改 `extract_functions` 使用 `&str` 引用
- 优化 `FileLoc` 结构体，使用引用而非复制

### 步骤 4: 添加内存监控（可选）
- 添加 `--memory` 参数显示内存使用

- 减少 30-50% 的内存使用
- 支持扫描超过 1GB 的代码库
- 扫描速度提升 10-20%
